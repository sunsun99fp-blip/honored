# Course Roadmap

This class is about finishing a public brand website, not learning every web technology.

| Stage | id | Done means |
|---|---|---|
| 1. 준비 | `prepare` | Student has an AI coding tool account, project idea or brief, and a project folder. |
| 2. 로컬 구현 | `local` | Project files exist locally: `package.json`, `index.html`, `src/`, or app files. |
| 3. 미리보기 | `preview` | Student has opened the site locally through a dev server or browser preview. |
| 4. 내용/디자인 수정 | `content` | Main sections, copy, visual tone, and mobile layout have had at least one pass. |
| 5. Git 저장 | `git` | `.git` exists and at least one commit exists. |
| 6. GitHub 연결 | `github` | `origin` remote exists and points to GitHub. Prefer confirming push succeeded. |
| 7. Vercel 배포 | `vercel` | Vercel project is connected or a public Vercel URL is recorded. |
| 8. 공개 URL 확인 | `qa` | Public URL opens, mobile layout is acceptable, and main links or CTA are checked. |
| 9. 다음 확장 | `next` | Supabase, DB, auth, API, and admin features are parked for after class. |

## Default Next Actions

- `prepare`: Write a 5-line brief: target audience, desired impression, sections, color mood, CTA.
- `local`: Ask the coding agent to create the first page from the brief.
- `preview`: Run the dev server, open the local URL, and screenshot anything broken.
- `content`: Fix the biggest visual issue first, then copy, then mobile.
- `git`: Run `git status`, commit the first working version.
- `github`: Create a GitHub repository, add remote, push.
- `vercel`: Import the GitHub repository into Vercel and wait for build logs.
- `qa`: Open the public URL on phone, check hero, sections, links, and CTA.
- `next`: List future features only; do not implement Supabase during this class.
