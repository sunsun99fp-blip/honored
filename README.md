# Honored · 시니어 여행 일정 플래너

50~60대 여행객이 인기 여행지부터 주변 맛집·카페·볼거리까지 한 번에 찾아
**나만의 날짜별 여행 일정표**를 만드는 사이트입니다.
로그인 없이 누구나 사용할 수 있고, 일정표는 브라우저에 저장되어 새로고침해도 유지됩니다.

## 주요 기능 (MVP)

- 🏯 인기 여행지 6곳 둘러보기 (경주·전주·강릉·안동·여수·제주)
- 📍 여행지별 주변 맛집·카페·볼거리 보기 + **담기**
- 🗓️ 담은 장소를 **날짜별로 배치 / 순서 변경 / 날짜 이동**
- 💾 **localStorage 저장** — 로그인 없이 새로고침 후에도 유지
- 📋 일정표 **텍스트로 복사·공유**
- 📱 모바일 / 데스크톱 모두 대응, 시니어 친화 큰 글자 디자인

## 폴더 구조

```
.
├── index.html          첫 화면(뼈대)
├── css/styles.css      디자인 (다크 그린 + 화이트 세리프)
└── js/
    ├── data.js         여행지·장소 mock 데이터 (콘텐츠 계층)
    ├── store.js        일정표 저장소 (localStorage → 나중에 Supabase 교체 지점)
    └── app.js          화면 렌더링 + 라우팅 (UI 계층)
```

데이터(`data.js`) · 저장(`store.js`) · 화면(`app.js`)을 분리해 두어,
나중에 Supabase 로 옮길 때 **`store.js` 한 파일만** 고치면 됩니다.

## 로컬에서 보기

빌드가 필요 없습니다. 두 가지 방법 중 하나면 됩니다.

1. **가장 간단:** `index.html` 파일을 더블클릭해서 브라우저로 엽니다.
2. **권장(개발 서버):** 폴더에서 아래 중 하나 실행 후 안내된 주소 접속
   ```bash
   npx serve .
   # 또는
   python -m http.server 5500
   ```

## Vercel 배포

이 프로젝트는 **정적 사이트**라 설정이 필요 없습니다.

1. 이 폴더를 GitHub 저장소로 푸시합니다.
2. [vercel.com](https://vercel.com) → **New Project** → 저장소 선택.
3. Framework Preset: **Other**, Build Command: 비움, Output Directory: `.` (기본값) → Deploy.
4. 만들어진 공개 URL을 **모바일·데스크톱**에서 확인합니다.

> 환경변수: 없음 (Supabase 미사용)

### 배포 전 체크리스트

- [ ] 모바일 첫 화면에서 메시지와 “여행지 둘러보기” CTA가 잘 보이는가
- [ ] 여행지 → 상세 → “담기” → 일정표까지 끊김 없이 동작하는가
- [ ] 담은 장소가 없을 때 빈 상태 화면이 안내되는가
- [ ] 새로고침 후에도 일정표가 유지되는가
- [ ] 일정표 “복사하기”가 동작하는가
